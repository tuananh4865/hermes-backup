**Giải pháp bất ngờ:**

**Kỹ thuật đằng sau (--mmap):**
`--mmap` bật memory-mapped file access. Thay vì nạp toàn bộ model (35B tham số) vào RAM, llama.cpp sẽ **memory-map file model vào RAM** — nghĩa là chỉ đọc phần nào cần thiết từ disk mà không phải nạp tất cả một lúc. macOS tự động quản lý phần nào được cache trong RAM, phần nào giữ trên disk.

**Kết quả thực tế:**
- Mac Mini M4 base ($599, 16GB) + llama.cpp + `--mmap` = chạy được 35B model
- Zero swap (không dùng disk làm virtual RAM)
- Tiết kiệm chi phí cực lớn so với việc dùng cloud API
