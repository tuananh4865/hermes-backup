## VIỆC 2: Công cụ browser tự động không tốn token + gemma4 vision

**Câu trả lời ngắn:** Dùng **browser-use** (open source) + **gemma4 e2b** (local, free) là combo tốt nhất cho Mac M4.

**Kiến trúc:**

```
┌─────────────────────────────────┐
│  Playwright (browser điều khiển)│  ← tự động click, scroll, type
└──────────────┬──────────────────┘
               │ chụp ảnh màn hình mỗi action
               ▼
┌─────────────────────────────────┐
│  gemma4 e2b (Mac M4 GPU)       │  ← phân tích ảnh, quyết định hành động
│  LM Studio: 192.168.0.187:1234  │  ← vision model miễn phí, local
└──────────────┬──────────────────┘
               │ hành động ("click vào nút đăng nhập")
               ▼
┌─────────────────────────────────┐
│  Playwright thực thi lệnh       │
└─────────────────────────────────┘
```

**Tại sao combo này tốt cho Mac M4:**
- gemma4 e2b: hỗ trợ vision (nhìn màn hình), chạy local trên Mac M4 GPU — **không tốn token**
- browser-use: Playwright wrapper mạnh nhất hiện nay, element targeting chính xác
- fast-whisper: nếu cần transcribe nội dung trên trang — ghép thêm

**Cài đặt:**

```bash
# 1. Clone browser-use
cd /tmp && git clone https://github.com/browser-use/browser-use.git

# 2. Cấu hình dùng local LM Studio thay vì OpenAI
# Trong code: thay api_key bằng LM Studio endpoint
```

**Lưu ý quan trọng mà em phát hiện lần trước:** gemma4 e2b trên Ollama website ghi là "Audio supported" nhưng thực tế **chỉ hỗ trợ vision, không hỗ trợ audio**. Vậy cho browser automation thì perfect — chỉ cần screenshot → gemma4 phân tích → quyết định.

**Hoặc:** Dùng luôn tool browser có sẵn trong Hermes (`browser_navigate`, `browser_snapshot`, `browser_click`) — những tool này đã dùng Playwright bên dưới rồi. Muốn nâng cấp thêm bước "AI phân tích màn hình rồi quyết định click chỗ nào" thì mới cần gemma4 vision.
