# Retrospectives

> **Purpose**: Session-based lessons and reflections. After each work session, capture what happened.

## Format
Each retrospective captures:
- **What we accomplished** — completed items
- **What went wrong** — issues, bugs, time sinks
- **Lessons to remember** — insights for next session
- **Next actions** — concrete steps for next time

## Philosophy

1. **End of session = learning moment** — Capture while fresh
2. **Atomic entries** — One session = one note
3. **Link mistakes** — If mistake happened, log it in mistake log

## When to Write Retrospective

- After any work session where progress was made
- After debugging session (especially if > 30 min)
- After code review with feedback
- Weekly check-in for long projects

## Session Types

### Daily Standup Retrospective
Quick check-in for active projects. Focus on:
- What's done
- What's blocked
- What's next

### Weekly Retrospective
Review week progress. Focus on:
- Accomplishments vs goals
- Blockers that persisted
- Adjustments for next week

### Phase Retrospective
After phase completion. Focus on:
- Phase goals achieved?
- What slowed us down?
- What would we do differently?
