# JSON

JSON (JavaScript Object Notation) is a lightweight, text-based data interchange format designed for representing structured data. It was introduced in the early 2000s as a simpler alternative to [[XML]] and quickly became one of the most widely used data formats on the web. JSON is language-independent, meaning it can be parsed and generated by virtually any programming language, making it the de facto standard for data exchange between heterogeneous systems.

Originally derived from JavaScript, JSON is now used across a vast range of applications, from [[REST API]] responses and configuration files to database storage and message queues. Its simplicity, readability, and universal support have made it the preferred choice for modern web services and [[web application]] development.

## Overview

JSON was created by Douglas Crockford in the early 2000s as a way to transmit data between a server and a web page. It was formalized in RFC 4627 in 2006 and later updated in RFC 7159 and ECMA-404, which established it as an official international standard.

The format was designed to be minimal, portable, and easy to read. Unlike [[XML]], which uses verbose tag-based markup, JSON represents data using a compact key-value structure that maps directly to native data structures in most programming languages, such as objects, arrays, strings, numbers, and booleans.

JSON is widely used in [[AJAX]] applications for asynchronous data loading, in [[NoSQL]] databases like MongoDB and CouchDB for document storage, and as the primary serialization format for [[REST API]] communications. It is also the foundation for [[JSON Web Token]] (JWT), a standard for securely transmitting claims between parties.

## Syntax

JSON data is organized into two primary structures: objects and arrays. A JSON object is an unordered collection of key-value pairs, enclosed in curly braces `{}`. Each key must be a string, and values can be strings, numbers, booleans, null, objects, or arrays.

A JSON array is an ordered list of values, enclosed in square brackets `[]`. Values within an array can be of any valid JSON type, including other objects and arrays, enabling the creation of complex nested data structures.

JSON strings must be enclosed in double quotes, and special characters within strings are escaped using a backslash. Comments are not permitted in standard JSON, which contributes to its simplicity but can be a limitation during development and debugging.

Here is a simple example of a JSON object representing a user:

```json
{
  "name": "Alice Johnson",
  "age": 30,
  "email": "alice@example.com",
  "is_active": true,
  "roles": ["admin", "editor"],
  "address": {
    "city": "San Francisco",
    "zip": "94102"
  }
}
```

This example demonstrates the key features of JSON syntax: string keys, various data types, nested objects, and arrays. The structure is self-describing and can be directly mapped to native data structures in programming languages like Python dictionaries or JavaScript objects.

## vs XML

JSON and [[XML]] are both formats for representing structured data, but they differ significantly in design philosophy, syntax, and use cases. Understanding these differences helps developers choose the appropriate format for their specific needs.

**Verbosity**: XML uses opening and closing tags for every element, making it considerably more verbose than JSON. A simple data record that fits in a few lines of JSON can require many more lines in XML. This verbosity affects bandwidth consumption and parsing speed, particularly in resource-constrained environments.

**Data typing**: JSON has native support for basic data types including strings, numbers, booleans, null, arrays, and objects. XML treats all content as text by default, requiring explicit schemas or namespaces to define data types. JSON's native typing reduces the need for additional parsing logic.

**Readability**: While both formats are human-readable, JSON is generally considered more accessible due to its compact syntax. XML's hierarchical nesting with tags can become difficult to read with deeply nested structures. JSON's key-value notation maps closely to how data is naturally conceptualized.

**Parsing complexity**: JSON can be parsed natively by JavaScript using the `JSON.parse()` method, making it extremely efficient in web browsers. XML requires the Document Object Model (DOM) API for parsing, which is more complex and slower. This difference has contributed to JSON's dominance in web development.

**Metadata handling**: XML excels at handling metadata and document-centric data through attributes, namespaces, and schemas. It supports comments, which JSON does not. For applications requiring extensive metadata or complex document structures, XML may still be the better choice.

**Ecosystem support**: While XML remains important in enterprise systems, SOAP web services, and document formats like Microsoft Office files, JSON has become the dominant format for [[REST API]] and [[web application]] development. Most modern APIs preferentially support JSON, and tools like [[Swagger]] and [[OpenAPI]] specifications are built around JSON.
