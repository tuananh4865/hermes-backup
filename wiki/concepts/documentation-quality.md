# Documentation Quality

High-quality documentation is essential for knowledge sharing, onboarding, and maintaining complex systems over time.

## Principles

- **Clarity**: Write for the target audience, avoiding jargon where possible
- **Completeness**: Cover all essential aspects without unnecessary verbosity
- **Accuracy**: Verify information and keep docs updated as systems evolve
- **Discoverability**: Structure content for easy navigation and search

## Documentation Types

- **Reference**: Detailed API/command specifications
- **Tutorials**: Step-by-step learning guides
- **How-to Guides**: Task-oriented problem solving
- **Explanations**: Conceptual understanding and background

## Best Practices

- Use consistent formatting and terminology
- Include code examples that can be tested
- Keep docs near the code (living documentation)
- Review and update regularly
