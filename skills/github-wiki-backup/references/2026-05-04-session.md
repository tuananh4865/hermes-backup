## When to Actually Commit Skills
Use `git status --short` BEFORE committing to detect real changes:
```bash
git status --short skills/ .gitignore  # Shows files only if real changes exist
```
If output is empty → skills had no changes, skip the commit.

## When to Actually Commit Memories
```bash
git diff --stat .hermes/memories/  # Shows stat only if real diffs exist
```
If no output → memories had no changes, skip the commit.

## Anti-Pattern Observed
The original cron job prompt runs ALL THREE stages unconditionally and pushes separately for each. The correct pattern is: check → conditional commit → push. Unconditional commits of unchanged content waste GitHub history with noise commits.
